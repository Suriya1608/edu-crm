4. Escalation Matrix
Automated escalation system that notifies managers when SLAs are breached or follow-ups are missed
Escalations are triggered automatically by the AutomationEngine service. They run when the Manager dashboard loads, and can also be scheduled via a command.

Escalation Type 1 — Missed Follow-up Escalation

Purpose: Alert the manager when a telecaller had a scheduled follow-up but didn't act on it.

Trigger Conditions

  • A follow-up's next_followup date is in the past (today or earlier)
  • The follow-up has not been escalated yet (escalated_at IS NULL)
  • The assigned telecaller had no recorded activity on or after the scheduled date (no call, WhatsApp, note, status change, or follow-up log)
  • Automation setting: Escalate Missed Follow-ups = ON
  • Current time is within business hours (if business hours enforcement is enabled)

What Happens

Follow-up date passes
→
No telecaller activity found
→
Notify manager (in-app)
→
Set escalated_at = now()

Notification Sent to Manager

Title: "Missed Follow-up Escalation" Message: "Telecaller {name} missed follow-up for lead {code} on {date}." Link: Lead detail page (manager panel) Type: MissedFollowupEscalationNotification Max 100 escalations per run to prevent overload

Escalation Type 2 — Response SLA Breach

Purpose: Alert the manager when a new lead hasn't been contacted within the configured SLA window.

Trigger Conditions

  • Lead status is new or open
  • Lead was created more than X minutes ago (X = SLA setting, default 60 mins)
  • Lead has not been escalated yet (sla_escalated_at IS NULL)
  • Lead has no recorded activity of any type (no call, WhatsApp, note, follow-up, or status change)
  • Lead has an assigning manager (assigned_by IS NOT NULL)
  • Automation setting: Escalate Response SLA = ON

What Happens

Lead created
→
SLA time passes (e.g. 60 min)
→
No activity found
→
Notify manager (in-app)
→
Set sla_escalated_at = now()

Notification Sent to Manager

Title: "SLA Breach — Lead Not Contacted" Message: "Lead {code} ({name}) has not been contacted in {X} minutes." Link: Lead detail page (manager panel) Type: SlaViolationEscalationNotification Max 100 escalations per run

5. Follow-up Notification System
How reminders reach telecallers — in-app bell, badge count, and sound alerts

How Reminders Are Triggered

The notification system works through a polling architecture. The telecaller's browser polls the server every 20 seconds to check for new notifications. Each poll also triggers the automation engine to dispatch any pending reminders — so reminders go out the moment they're due, even without a background job.

Polling Flow

Browser polls every 20s
→
AutomationEngine runs
→
Pending reminders sent
→
Response returned to UI
→
Bell badge + sound

Notification Categories

CategoryConditionLimitData Shown
📞 Missed Calls Inbound calls with status = missed or no-answer, assigned to this telecaller Top 5 Lead name, code, phone, time, link to lead
📅 Follow-up Reminders Follow-ups with next_followup ≤ (today + N days), assigned_to = telecaller, not completed Top 5 Lead name, code, date, type (overdue/today/upcoming), remarks
🔔 System Notifications Unread entries in the Laravel notifications table for this user Top 5 Message text, relative time (e.g. "5 minutes ago")

Follow-up Reminder Dispatch Process

Each time the notification snapshot endpoint is called, AutomationEngine::dispatchTelecallerFollowupReminders() runs:

StepActionDetail
1Check if enabledReads follow_up_reminder_enabled setting; exits if off
2Check business hoursOnly runs if within operational window (if enforcement is on)
3Query follow-upsnext_followup between today and (today + days_before), reminder_notified_at IS NULL, completed_at IS NULL
4For each follow-upGet telecaller (via lead.assigned_to)
5Send notificationCreates a database notification of type SlaViolationEscalationNotification
6Mark as notifiedSets reminder_notified_at = now() so reminder is not sent again

Notification UI — Telecaller Bell Icon

Badge Count

  • Red badge on bell icon in header
  • Count = sum of missed calls + follow-up reminders + system notifications
  • Hides automatically when count = 0
  • Updates every 20 seconds

Sound Alert

  • Plays a beep when badge count increases
  • Telecaller can toggle sound on/off
  • Preference saved in localStorage (persists across sessions)

Notification Types Display

  • Overdue — overdue red badge
  • Due today — today orange badge
  • Upcoming — upcoming blue badge

Mark All Read

  • Button in notification dropdown
  • Marks missed calls and follow-ups as "seen" in localStorage
  • Marks database notifications as read
  • Badge resets to 0

Follow-up Reminder Configuration

Remind N Days BeforeHow many days in advance to remind telecaller0 – 30 days0 (same day)

Example: If "Remind 2 Days Before" is set, a follow-up due on March 5 will appear in the telecaller's notification panel starting March 3.

7. Enterprise Enhancements (March 2026)
Queue-backed automation, audit logs, policy security, call outcomes, lead aging, and more

Queue-Backed Automation

Automation no longer runs synchronously on HTTP requests. All automation is dispatched via Laravel queue jobs, scheduled through routes/console.php.

ComponentClassSchedule
Escalation EngineJobs\DispatchEscalationsEvery minute (via crm:run-automation)
Follow-up RemindersJobs\DispatchFollowupRemindersEvery minute (via crm:run-automation)
Daily Summary EmailJobs\SendDailySummaryDaily at 19:00 (via crm:daily-summary)
Each job uses Cache::lock() with a 5-minute TTL to prevent overlapping runs. Run php artisan schedule:run every minute via Task Scheduler to activate scheduling.

Audit Log System

AuditLogService::log() records all sensitive operations to the audit_logs table. It never throws exceptions — silent failure protects user-facing requests.

ActionTrigger
lead.assignedManager assigns telecaller to a lead
lead.status_changedAny role changes lead status
lead.duplicate_detectedDuplicate phone on create or import
lead.mergedAdmin merges two duplicate leads
user.updatedAdmin edits a user profile
user.status_changedAdmin toggles user active/inactive
user.force_logoutAdmin force-logs out a user

Laravel Policies (Role Security)

PolicyGovernsRules
LeadPolicyLead resourceAdmin = all; Manager = own team; Telecaller = assigned leads only
UserPolicyUser managementAdmin only
SettingPolicySystem settingsAdmin only
Gate::before() grants admin users all abilities automatically — no per-ability check needed for admins.

Duplicate Lead Handling

On Create
  • Phone normalized (digits only) before check
  • If phone exists: is_duplicate = true
  • Lead still created (not blocked)
  • Warning shown in success message
  • Audit log entry created
On Import
  • Batch duplicate detection (O(1) via array flip)
  • Within-batch duplicates also flagged
  • Count shown: "47 imported, 3 flagged"
  • Per-duplicate audit log entry
Merge (Admin)
  • Route: POST /admin/leads/{id}/merge/{targetId}
  • Moves activities, followups, call logs, WhatsApp messages
  • Source marked status = merged
  • Merge button on Duplicate Leads view
  • Audit log: lead.merged

8. Outbound Bulk Campaign Module
Upload bulk student databases, auto-distribute to telecallers, and track outbound calling campaigns — completely separate from the Lead Capture system
This module is fully independent from the Lead Capture Management system. It uses its own database tables (campaigns, campaign_contacts, campaign_activities) and does not modify any existing lead, followup, or call log records.

Overview

The Campaign Module enables managers to run outbound calling campaigns — for example, uploading a database of 12th-pass students and distributing them among telecallers who will then call and promote admissions.

Manager Features

📋 Campaign Management
  • Create named campaigns with description
  • Campaign statuses: Active, Paused, Completed, Draft
  • Change campaign status from the detail page
  • View all campaigns with contact counts
📂 Bulk Upload (Excel/CSV)
  • Upload .xlsx, .xls, or .csv files
  • Expected columns: Name | Mobile | Email | Course | City
  • Preview records before confirming import
  • Duplicate detection: skips records with same phone or email within the same campaign
  • Import summary: Total / Inserted / Duplicates skipped
🔀 Auto-Distribution
  • Select one or more telecallers to distribute to
  • Round-robin equal distribution (e.g. 1000 contacts ÷ 5 telecallers = 200 each)
  • Distributes only unassigned contacts
  • Shows online/offline telecaller status
  • Assignment summary panel per campaign
📊 Campaign Analytics
  • Total contacts, Pending, Contacted, Interested, Converted counts
  • Filter contacts by status, telecaller, or search (name/phone/email)
  • View all contacts with status badges and next follow-up dates
  • Upload more contacts to an existing campaign at any time

Telecaller Features

📱 My Campaigns
  • See all campaigns with contacts assigned to them
  • View contact count per campaign
  • Filter contacts by status or search by name/phone
  • Paginated contact list (20 per page)
📞 Contact Actions
  • Click-to-call button — triggers global call bar
  • WhatsApp button — opens WhatsApp Web with pre-filled message
  • Update call status (Pending / Called / Interested / Not Interested / No Answer / Callback / Converted)
  • Set follow-up date (date picker, future dates only)
  • Add text notes
📅 Activity Timeline
  • Full activity history per contact — calls, WhatsApp, notes, status changes, follow-ups
  • Filter timeline by type: All / Calls / WhatsApp / Notes
  • Call count tracked on each contact
  • Auto-marks status as "Called" on first call if still Pending
  • Same visual timeline design as Lead Show page
